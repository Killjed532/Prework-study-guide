HTML

Basic HTML Syntax

   Elements:

  • Content is contained between an opening tag (<head>) and closing tag (</head>).
  • Other common elements are <head>, <title>, <body>, <header>, <footer>, <article>, <section>, <p>, <div>, <span>, <img>, <aside>, <audio>, <canvas>, <datalist>, <embed>, <details>, <nav>, <output>, <progress>, <video>, <ul>, <ol>, <li>, and many others.

How to Add <> to HTML text

  • For < type &lt;
  • For > type &gt;

Attributes

  • Attributes are contained in the opening tag. Attributes contain extra information about the element that you don't want to appear in the actual content.
  • Some common attributes are href, src, width, height, title, style, alt, lang, class. id, and many more.
* If an element is not linking correctly to a webpage there is likely a problem with the attributes. Give them a second look to make sure they're correct.

<head>

  • The head element contains information about the webpage.
  • In this section a developer would add the following:
    • <link> to link to an external file
      • The rel attribute specifies the relationship between the current document and the linked document.
      • The href attribute specifies the location of the external resource.
    • <meta> this would contain the author, keywords, a description, and information to assist a browser in rendering the page correctly.

<body>

  • The body element represents the visible content shown to the user.
  • The body contains three child elements. They are as follows:
    1. <header>
      • The header element represents introductory content, typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements.
    2. <main> element represents the dominant content of the <body> of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.
      • This is where the bulk oof the content on the page will go.
    3. <footer> element represents a footer for its nearest ancestor sectioning content or sectioning root element.
      • This appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation

Other Common Elements

  • <h1-6> these are used as headings the lower the number the larger the text.
  • <p> p stands for a paragraph.
  • <span> is used to create inline division.
  • <div> is used to create a division in the page.
  • <aside> element represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes.
  • <br> this forces a new line in the block element
  • <title> element defines the document's title that is shown in a browser's title bar or a page's tab. It only contains text; tags within the element are ignored.
  • <article> element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable.
  • <section> element represents a generic standalone section of a document, which doesn't have a more specific semantic element to represent it. Sections should always have a heading, with very few exceptions.
  • <audio> element is used to embed sound content in documents. It may contain one or more audio sources, represented using the src attribute or the <source> element: the browser will choose the most suitable one./li>
  • <canvas> use this element with either the canvas scripting API or the WebGL API to draw graphics and animations.
  • <datalist> element contains a set of <option> elements that represent the permissible or recommended options available to choose from within other controls.
  • <embed> element embeds external content at the specified point in the document. This content is provided by an external application or other source of interactive content such as a browser plug-in.
  • <details> element creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state. A summary or label must be provided using the <summary> element.
  • <nav> element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples of navigation sections are menus, tables of contents, and indexes.
  • <output> element is a container element into which a site or app can inject the results of a calculation or the outcome of a user action.
  • <progress> element displays an indicator showing the completion progress of a task, typically displayed as a progress bar.
  • <video> element embeds a media player which supports video playback into the document. You can use <video> for audio content as well, but the <audio> element may provide a more appropriate user experience.

List Elements

CSS

Basic CSS

What is CSS?

  • CSS is a stylesheet language used to describe the presentation of a document written in HTML or XML.
  • CSS describes how elements should be rendered on screen, on paper, in speech, or on other media.

CSS Ruleset

  • Selector indicates the element you wish to work on
  • Declaration is the combination of a property and a property value.
  • Property chooses the type of property you wish to influence or change of the selector.
  • Property value is how much or little you want to change the property.
**Each ruleset must be contained between curly braces {}, there must be a colon (:) between to property and property value, and each declaration must end with a semicolon (;)

Different Types of Selectors

  • Element Selector
    • This selects all HTML elements of the specified type.
  • ID Selector
    • The element on the page with the specified ID.
  • Class Selector
    • The element(s) on the page with the specified class.
  • Attribute Selector
    • The element(s) on the page with the specified attribute.
  • Pseudo-Class Selector
    • The specified element(s), but only when in the specified state. (For example, when a cursor hovers over a link.)

Specificity

   These selectors give a score to the ruleset that is attempting to be implemented. The high the score indicates which rule will be applied. The specificity scoring is broken down into three categories (ID 0, Class 0, Type 0).

Box Styling

  • margin is the space around the border.
  • padding is the space around the element between the border and the element
  • border is the border around the element. To create a border choose a thickness, a border type, and color.
  • box-shadow is the shadow of the box.
    • /* offset-x | offset-y | blur-radius | spread-radius | color */
    • box-shadow: 2px 2px 2px 1px rgba(0, 0, 0, 0.2);

Fonts and Text

   It is possible to link to Google fonts either in the <head> of the HTML or to simply import them at the beginning of the CSS document. Apart from importing the font style in CSS you can change the following and many more:

  • font-size
  • font-weight
  • font-style
  • text-indent
  • text-align

Coloring

   It is possible to change the background color of an element using background-color in the property section. There are few different methods to get a color hex, rgb oe rgba, and hsl. both text and background can be changed in the different elements. Be sure that the foreground and background are readable.

Manipulating a Image or Box

   To center an image type the property {display: block; margin: 0, auto;} when there are two property values in margin the first is for top and bottom and the second is for left and right. There are also options to change the placement of boxes by doing a display: flex; or display: grid;. This will break a element that has other nested elements in it to be placed in a formation that can be manipulated. For example, display: flex; flex-flow: row nowrap; justify-content: space-evenly;.

Basics of Git

  • Clone a repository

    To clone a repository, use the command: git clone [repository url]

  • Check the status of your repository

    To check the status of your repository, use the command: git status

  • Add changes to the staging area

    To add changes to the staging area, use the command: git add [file name]

  • Commit changes

    To commit changes, use the command: git commit -m "[commit message]"

  • Push changes to a remote repository

    To push changes to a remote repository, use the command: git push [remote name] [branch name]

  • Pull changes from a remote repository

    To pull changes from a remote repository, use the command: git pull

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it

JavaScript

Basics of JavaScript

Basic JS Syntax

   JavaScript is a scripting language that is primarily used for client-side web development. The basic syntax of JavaScript includes statements, variables, and functions. JavaScript statements are executed by the browser, and are often used to change the content or style of an HTML or CSS document. Variables are used to store data, and functions are used to perform specific tasks.

Variables

   In JavaScript, variables are used to store data. A variable is declared using the keyword "var", "let" or "const" followed by the variable name. The variable name should start with a letter or underscore, and can contain letters, numbers, and underscores. For example: var x = 5; or let y = "Hello";

Objects

   In JavaScript, objects are used to store multiple values in a single variable. Objects are created using curly braces {} and properties and values are separated by a colon :. For example: let person = {name: "John", age: 30};

Strings

   JavaScript strings are used to store and manipulate text. Strings are written within quotation marks, either single or double. For example: let name = "John Doe";

Arrays

   JavaScript arrays are used to store multiple values in a single variable. There are different types of arrays like simple arrays, Multidimensional arrays, Jagged arrays. For example: let fruits = ["apple", "banana", "orange"];

Integers

   JavaScript integers are whole numbers without a decimal point. For example: let x = 5;

Loops

   JavaScript loops are used to execute the same code multiple times. JavaScript supports for, while, do-while and for-of loops. For example: for (let i = 0; i < 5; i++) { console.log(i); }

Booleans

   JavaScript booleans are used to represent true or false values. For example: let x = true; or let y = false;

Numbers

   JavaScript numbers are used to represent numeric values. JavaScript supports both integers and floating-point numbers. For example: let x = 5; or let y = 3.14;

Comments

   JavaScript comments are used to add notes or explanations to your code. Single-line comments start with // and multi-line comments start with /* and end with */. For example: // This is a single-line comment or /* This is a multi-line comment */

Data Structures

  • Stack: A stack is a data structure that follows the Last In First Out (LIFO) principle. It allows elements to be added and removed only from the top of the stack. This means that the last element added to the stack will be the first one to be removed.
  • Queue: A queue is a data structure that follows the First In First Out (FIFO) principle. It allows elements to be added at the rear and removed from the front. This means that the first element added to the queue will be the first one to be removed.
  • Linked List: A linked list is a linear collection of data elements, called nodes, where each node points to the next node in the list. The first node is called the head, and the last node is called the tail.
  • Set: A set is a collection of unique elements. It does not allow duplicate values and does not have any specific order.
  • Hash Table: A hash table is a data structure that stores data in an array-like format, but with the ability to quickly search for a specific element using a key. It uses a hash function to map keys to positions in the array, allowing for fast lookups and insertions.
  • Tree: A tree is a data structure that is composed of nodes, where each node can have one or multiple child nodes. The topmost node is called the root, and the nodes that do not have any child nodes are called leaf nodes.
  • Trie: A trie is a tree-like data structure that is used to store an associative array where the keys are sequences (usually strings). It is also known as a prefix tree.
  • Graph: A graph is a data structure that is composed of vertices (also called nodes) and edges. Vertices represent the objects in the graph, and edges represent the relationships between them. Graphs can be directed (where edges have a specific direction) or undirected (where edges have no direction).
